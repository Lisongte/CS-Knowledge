# CS-Knowledge

我的 CS 知识库：只存可复用的理解、代码片段、踩坑结论和项目复盘。原则：编号稳定、路径清楚、短句高密度。

## 0. 导航规则

编号格式：`大类.子类.主题.条目`，不补零。

- `0`：仓库元信息、学习路线、复盘
- `1`：编程语言与工程实践
- `2`：数据结构与算法
- `3`：机器学习、Kaggle、数据分析
- `4`：计算机系统
- `5`：计算机网络
- `6`：数据库
- `7`：项目

例子：`3.1.1` = 机器学习 / Kaggle / 读取训练集 CSV。

## 3. 笔记格式

每篇笔记尽量压成 5 块：

```md
# 编号 - 标题

## 一句话

## 最小代码

## 关键规则

## 常见坑

## 记忆钩子
```

目标：少写“我应该怎么学”，多写“我现在知道了什么”。
